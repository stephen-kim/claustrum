# 부트스트랩 관리자 설정

Claustrum은 깨끗한 초기 설치를 위한 첫 실행 부트스트랩 관리 흐름을 지원합니다.

## 초기 계정

- 첫 번째 부트스트랩에서 이메일이 수정되었습니다: `admin@example.com`
- 부트스트랩은 `users` 테이블이 비어 있는 경우에만 실행됩니다.
- 서버는 초기 비밀번호를 서버 로그 스트림에 한 번 인쇄합니다.

예제 출력:

```text
Bootstrap admin created: admin@example.com
Initial password (shown once): <random-password>
```text
## 첫 번째 로그인 요구 사항

부트스트랩 자격 증명으로 로그인한 후 플랫폼을 사용하기 전에 설정을 완료해야 합니다.

1. 이메일 변경(필수, `admin@example.com` 유지 불가)
2. 비밀번호 변경(필수)
3. 표시 이름 설정(선택 사항)

설정이 완료될 때까지:
- `/v1/auth/me`, `/v1/auth/complete-setup`, `/v1/auth/logout`이 허용됩니다.
- 다른 `/v1/*` API는 `403`으로 차단됩니다.

## 재설치/재설정 동작

- DB가 재설정되고 `users`이 다시 비어 있으면 부트스트랩이 다시 실행되어 새로운 일회용 비밀번호를 인쇄합니다.
- 사용자가 이미 존재하는 경우 부트스트랩이 실행되지 않으며 비밀번호가 인쇄되지 않습니다.

## 보안 권장 사항

- 부트스트랩 비밀번호 출력을 민감한 비밀 자료로 처리합니다.
- 실제 개인 비밀번호로 즉시 회전합니다.
- 보안 로그 싱크를 선호하고 시작 로그를 공개적으로 노출하지 마세요.
